# Large-Scale Instances

This folder contains 9 large-scale SCFLP instances generated by the authors following a similar procedure as described in Qi et al. (2024). See Section 7.2 of the paper for details.

The locations of demand points and candidate facility sites are randomly generated with integer coordinates on a [0,70]×[0,70] Euclidean plane. The attractiveness parameter is set to v_ij = exp(-0.1 * d_ij) based on the partially binary rule, where d_ij is the Euclidean distance between demand point i and facility site j. Customer demands are chosen uniformly at random from {1,...,10}.

The 9 files cover all combinations of m ∈ {500, 800, 1000} and n ∈ {500, 800, 1000}, where m is the number of demand points and n is the number of candidate facility sites. Each instance is tested with p, r ∈ {2, 5, 10, 50} (16 combinations per file, 144 instances in total).

## File Naming

`instance_<m>_<n>.csv` where `m` = number of demand points, `n` = number of candidate facility sites.

## File Format

```
<m>,<n>, # demand points, # candidate sites
<x_1>,<y_1>,<w_1>
<x_2>,<y_2>,<w_2>
...
```

- The first line is a header specifying m and n.
- Each subsequent line gives the coordinates and demand weight of one point. The first m lines are demand points; the next n lines are candidate facility sites.
- `x`, `y`: integer coordinates in the Euclidean plane
- `w`: customer demand (weight), an integer in {1,...,10}

## Reference

> **Qi M, Jiang R, Shen S (2024)** Sequential competitive facility location: Exact and approximate algorithms. Oper. Res. 72(1):300–316.
